让视图控制器的视图位置不要和导航栏重合的三种方法

发表于 2016-06-24

方法1 设置edgesForExtendedLayout

rootViewController.edgesForExtendedLayout = UIRectEdgeNone;

viewController的左上角位置会在导航栏下面。

方法2 设置导航栏不透明

navigationController.navigationBar.translucent = NO;

viewController的左上角位置会在导航栏下面，会影响其导航控制器的其他子视图也下降。

方法3 所有的子视图都为viewController.view留白。

所有子视图都从y＝64开始算起。不把子视图放在y = 64更小的位置.看起来viewController.view变小了

这种方法最不可取。

导航栏透明的设置

发表于 2016-06-19

导航栏透明不能依靠alpha值

viewController.navigationController.navigationBar.alpha = 0.0;

一般视图都是利用alpha来设置透明度。这一套对于继承UIView的的navigationBar并不生效.设置后依然可以显示。实际上导航栏设置透明是只需设置NavigationBar的背景图片为一张空图片即可

[self.navigationController.navigationBar setBackgroundImage:[UIImage new] forBarMetrics:UIBarMetricsCompact];

透明的导航栏还可以放置titleView leftBarItem

在视图的概念中，如果设置alpha = 0后呈现透明效果后，其子视图也会透明。但是我们导航栏的透明并不是依赖｀alpha ＝ 0.而是依靠设置空白图片的方法。我们设置的titleView，leftBarItem依然可以显示。

UITableViewCell 动画

发表于 2016-06-19

目标与思路

希望做如上图的动画。观察这个动画，每一个cell从左到右进入眼帘。下面的cell比上面的cell到底最终位置的时间更久。现在有两种思路。
1.每个cell动画启动时间不同。动画的执行时间相同
2.每个cell动画启动时间相同。后面的动画执行时间更久。

方案2更简单，因为很难实现每一个动画的启动时间固定时间间隔

执行动画的时机

UITableView的委托方法来执行动画。

- (void)tableView:(UITableView *)tableView willDisplayCell:(UITableViewCell *)cell forRowAtIndexPath:(NSIndexPath *)indexPath

是tableViewCell的动画的最佳时机

该方法中可以使用[UIView animateWithDuration:animations:]。要注意的是，我们的动画只是在第一次cell出现在屏幕上的时候执行。在其他时候。如滑到上面后又滑下来的时候是不该被调用的。我们可以用一个全局变量。保证动画执行次数和屏幕的显示个数一致。

完整demo

//
//  ViewController.m
//  Animation
//
//  Created by zhou on 16/6/15.
//  Copyright © 2016年 jeff.zhou. All rights reserved.
//

#import "ViewController.h"
#import "ZZCell.h"

@interface ViewController () <UITableViewDelegate, UITableViewDataSource>
@property (nonatomic, strong) UITableView *tableView;

@property (nonatomic, assign) NSInteger count;
@end

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    [self setupTableView];
}

- (void)setupTableView {
    CGRect rect = CGRectMake(0, 0, 80, [UIScreen mainScreen].bounds.size.height);
    UITableView *tableView = [[UITableView alloc] initWithFrame:rect];
    tableView.delegate = self;
    tableView.dataSource = self;
    [tableView registerClass:[ZZCell class] forCellReuseIdentifier:@"cell"];
    self.tableView = tableView;
    self.count = 0;
    [self.view addSubview:self.tableView];
}

- (void)tableView:(UITableView *)tableView willDisplayCell:(UITableViewCell *)cell forRowAtIndexPath:(NSIndexPath *)indexPath {

    // 16是一屏显示的cell个数，这个方法是要保证第一次出现cell的时候动画
    if (self.count > 16) {
        return;
    }
    self.count++;

    CGPoint oldPoint = cell.frame.origin;
    CGPoint newPoint = oldPoint;
    newPoint.x = - 80 -indexPath.row * 80 * 0.15;
    // 设置开始位置
    cell.frame = CGRectMake(newPoint.x, newPoint.y, cell.frame.size.width, cell.frame.size.height);
    // 核心思想是越下面的cell离cell最后位置越远。每一个cell的动画时间相同。每个cell速度相同
    [UIView animateWithDuration:(0.5 + 0.125 * indexPath.row) animations:^{
     // 设置结束为止
        cell.frame = CGRectMake(oldPoint.x, oldPoint.y, cell.frame.size.width, cell.frame.size.height);
    }];
}

- (NSInteger)tableView:(UITableView *)tableView numberOfRowsInSection:(NSInteger)section {
    return 20;
}

- (UITableViewCell *)tableView:(UITableView *)tableView cellForRowAtIndexPath:(NSIndexPath *)indexPath {
    ZZCell *cell = [tableView dequeueReusableCellWithIdentifier:@"cell" forIndexPath:indexPath];
    return cell;
}


@end

Tip:一次git冲突文件的简单解决

发表于 2016-06-14

project.pbxproj的文件冲突如何解决？

今天使用pull代码的时候产生了project.pbxproj的文件冲突。按照惯常的方法打开project.pbxproj手动去搜索<<<<这样类似文本，删掉你不需要的，移动代码到需要的位置。但是实际过程中非常不理想。因为project.pbxproj的可读性比较差。因为记得自己本地的修改方法是创建了几个新的文件和目录。在同事的建议下，标记project.pbxproj为服务器的版本为准。然后这个时候再运行xcode,发现project.pbxproj不再报错。而是提醒找不到一些头文件。因为相关的头文件和实现文件虽然已经在了工程中。却没有被project.pbxproj所管理，这个时候只需要自己在工程所在的目录搜索后自己手动添加到xcode的管理中即可。

Tip:UIButton的imageView的显示

发表于 2016-06-11

场景

当我们在为视图控制器添加leftBarItem或者rightBarItem的时候，视觉工程师给我们的图的位置总不是苹果设计好的位置。而leftBarItem的customview又不是自己能决定的。这时候我们可以利用UIButton的一些属性来达到想要的位置和效果。

setImage:...与setBackgroundImage:...

使用setImage:会保持image大小不变，让内部的ImageView和image一样大，并将ImageView居中显示在button上
使用setBackgroundImage也会让image大小和button一样大，并且让imageview也和image，button一样。
总结来说。一个不会拉伸图像。一个会拉伸图像。但无论拉伸与否，都会让imageView的尺寸和image一样。
如图1，图2

imageEdgeInsets调整位置

UIEdgeInsets 是一个结构体,描述了一个视图与父视图的直接的间距。

typedef struct UIEdgeInsets {
    CGFloat top, left, bottom, right;  // specify amount to inset (positive) for each of the edges. values can be negative to 'outset'
} UIEdgeInsets;

正数表示离父视图内部的距离。负数表示离父视图外部的距离。top,bottom都是垂直维度的变量。left,right是水平维度的变量。我们只需要改变垂直维度的其中一个变量即可影响垂直位置。同理改变left就可以改变水平维度（right = 0 ） 。当我们使用setImage:设置imageView后，发现imageView水平位置不满足要求,可以这样设置

button.imageEdgeInsets = UIEdgeInsetsMake(28,0,0,0);

28将覆盖原来相对button垂直居中的布局。直接将让imageView垂直方向下与button间隔28.因为我们让top值为28.因为不再需要设置bottom.(bottom = 0)依然不变。注意这个时候不会对button进行拉伸。同时也不会改变水平位置。依然水平居中。就如图3。

如果要改变水平方向。可以设置

button.imageEdgeInsets = UIEdgeInsetsMake(0,28,0,0);

完整demo

#import "ViewController.h"

@interface ViewController ()

@end

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];

    {   // 不自动铺满，image维持原来的位置,并且居中
        UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
        button.frame = CGRectMake(20, 20, 44, 44);
        [button setImage:[UIImage imageNamed:@"add"] forState:UIControlStateNormal];
        button.backgroundColor = [UIColor redColor];
        [self.view addSubview:button];
    }

    {   // 自动铺满，不管uiimage 本身的大小
        UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
        button.frame = CGRectMake(20, 70, 44, 44);
        [button setBackgroundImage:[UIImage imageNamed:@"add"] forState:UIControlStateNormal];
        button.backgroundColor = [UIColor redColor];
        [self.view addSubview:button];
    }


//    UIKIT_STATIC_INLINE UIEdgeInsets UIEdgeInsetsMake(CGFloat top, CGFloat left, CGFloat bottom, CGFloat right) {

    {   // image维持原来的尺寸.
        UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
        button.frame = CGRectMake(20, 130, 44, 44);
        [button setImage:[UIImage imageNamed:@"add"] forState:UIControlStateNormal];
        button.backgroundColor = [UIColor redColor];

        // top 让他不再遵从垂直方向的居中。而是以button顶部内边距28
        // left,right等于0 表示依然遵从水平方向居中
        button.imageEdgeInsets = UIEdgeInsetsMake(28,0,0,0);
        [self.view addSubview:button];
    }


    {   // image维持原来的尺寸.
        UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
        button.frame = CGRectMake(20, 200, 44, 44);
        [button setImage:[UIImage imageNamed:@"add"] forState:UIControlStateNormal];
        button.backgroundColor = [UIColor redColor];
        // left 让他不再遵从水平方向的居中。而是以button左边内边距28
        //  注意left有值就不再是居中后再加上left的距离。而是直接以left距离为依据
        //  top,bottom= 0 表示依然遵从垂直方向居中
        //   在image"内部"插入一个你“看不见”的内边距
        button.imageEdgeInsets = UIEdgeInsetsMake(0,28,0,0);
        [self.view addSubview:button];

        UIView *blueView = [[UIView alloc] init];
        blueView.backgroundColor = [UIColor blueColor];
        blueView.frame = CGRectMake( 0, 14, 28, 16);
        [button addSubview:blueView];
    }

}

@end
