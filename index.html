Bash中命令连接符

每个命令被 && 号分隔，那么这些命令会一直执行下去，如果中间有错误的命令存在，则不再执行后面的命
command1 && command2 && command3

如果每个命令被双竖线(||)分隔符分隔，如果命令遇到可以成功执行的命令，那么命令停止执行，即使后面还有正确的命令则后面的所有命令都将得不到执行。
command1 || command2 || command3

函数

python中定义函数时无论函数是否需要返回，都不需要声明返回值

def my_abs(x):
    if x >= 0:
        return x
    else:
        return -x

类

类定义:类方法中，无论是初始化方法，还是其他方法，类方法必须包含参数self,且为第一个参数

def __init__(self, name, score):
   self.name = name
   self.score = score

MySQL数据库下载和安装

发表于 2016-07-14

命令行方式。。失败

下载

  1. 在浏览器中输入mysql.com
  2. 点击Download Tab,然后Download Tab下子分类下点击Community.
  3. 在打开的页面点击MySQL Community Server.
  4. 选择对应的平台，这里我选择Mac OS X, 选择第一个Mac OS X 10.11 (x86, 645. -bit), TAR,点击 Download
  5. 弹出页面要求你注册或者登陆，其实不用管它，注意左下角有一个No thanks, just start my download..点击 它，成功下载。

安装与简单配置

  1. 将下载的压缩包拷贝到/usr/local（在传统的unix系统中，/usr通常只包含系统发行时自带的程序，而/usr/local则是本地系统管理员用来自由添加程序的目录）下，我们使用命令行来完成
    sudo cp mysql-5.7.13-osx10.11-x86_64.tar.gz /usr/local

  2. 切换到/usr/local,解压目录

    cd /usr/local
    tar zxvf mysql-5.7.13-osx10.11-x86_64.tar.gz

  3. 解压后会有一个文件夹mysql-5.7.13-osx10.11-x86_64,将其重命名为mysql.
    mv mysql-5.7.13-osx10.11-x86_64 mysql

启动mysql

在启动mysql之前，需要执行一下初始化操作。卡在初始化操作中…下载dmg文件来操作。

总结

sudo cp mysql-5.7.13-osx10.11-x86_64.tar.gz /usr/local 
cd /usr/local
tar zxvf mysql-5.7.13-osx10.11-x86_64.tar.gz 
mv mysql-5.7.13-osx10.11-x86_64 mysql
rm mysql-5.7.13-osx10.11-x86_64

拷贝到指定目录，切换到该目录/usr/local,解压mysql,重命名mysql.删除压缩包

dmg方式 成功

下载

  1. 在浏览器中输入mysql.com
  2. 点击Download Tab,然后Download Tab下子分类下点击Community.
  3. 在打开的页面点击MySQL Community Server.
  4. 选择对应的平台，这里我选择Mac OS X, 选择最后一个Mac OS X 10.11 (x86, 64-bit), DMG Archive。
  5. 弹出页面要求你注册或者登陆，其实不用管它，注意左下角有一个No thanks, just start my download..点击 它，成功下载。

安装与简单配置

  1. 一路点击确定。最后会弹出一个框。告诉你应用程序给你的root账户分配了一个临时密码.

2016-07-14T16:14:55.804440Z 1 [Note] A temporary password is generated for root@localhost:QDWf-(f/2#x
If you lose this password, please consult the section How to Reset the Root Password in the MySQL reference manual.

2.点击左上角apple->preference.多出了一个MySql按钮，点击它，勾选start MySQL server。就启动了mysql 软件。

3.跳转到终端。输入mysql -u root -p.就是以root身份连接数据库。回车后要求输入密码。输入步骤1中弹出的密码。进入mysql命令行交互界面。mysql>

4.在交互界面中输入一个命令如，show databases.会弹出下面警告
ERROR 1820 (HY000): You must SET PASSWORD before executing this statement.此时，是因为刚才的root账号的密码是临时的账号密码。输入SET PASSWORD = PASSWORD('123456');就可以重置密码了。整个配置就完成了。

  • 注意在mysql交互界面，enter不会执行.要执行sql命令，必须使用
