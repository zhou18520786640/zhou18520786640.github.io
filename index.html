CAGradientLayer basic

发表于 2016-07-01

CAGradientLayer是用来生成两种或更多颜色平滑渐变的。

  1. 基础渐变

我们将从一个简单的红变蓝的对角线渐变开始，渐变色彩放在一个数组中，并赋给colors属性。这个数组成员接受CGColorRef类型的值（并不是从NSObject派生而来 ?），所以我们要用通过bridge转换以确保编译正常。
CAGradientLayer也有startPoint和endPoint属性，他们决定了渐变的方向。这两个参数是以单位坐标系进行的定义，所以左上角坐标是{0, 0}，右下角坐标是{1, 1}。

单位坐标系:无论frame大小多少，（0，0）表示左上角，（1，1）表示左下角，（1，0）表示左上角，（0，1）表示左下角。

iOS动画小结---UIView属性动画

发表于 2016-06-25

UIView的动画与Core Animation动画的关系

在iOS中，图形可分为以上几个层次：
越上层，封装程度越高，动画实现越简洁越简单，但是自由度越低；反之亦然。

UIKit的动画是最高层的动画，API最简单，实现自由度也最低。实现UIKit层的动画，一般流程需要通过以下两个条件。

1.可以被改变的动画属性
2.几个动画执行的类方法

Core Animation的动画一般通过类似[shapeLayer addAnimation:[CABasicAnimation animationWithKeyPath:@"...."] forKey:@"name"];，UIkit的动画实质是Core Animation的动画的封装。

可被执行的动画属性

我们先来看看可以被动画的属性，进入UIView的头文件。搜索animatable,可以看到下面这些属性被注释为animatable

alpha 透明度
frame 父视图坐标系统的位置
bounds 自身坐标系统的位置
center 中心点位置
transform 仿射变换

几个动画执行的类方法

动画的实现并不是在任何地方给可动画的属性进行赋值就可以了。只有将那些赋值方法作为动画类方法中的block传递过去才能实现动画。

起始点，终点，持续时间

执行动画基本流程。

  1. 对可动画的一个属性赋一个值。即设置可动画的属性的
  2. 在实现动画的视图类方法的block中给可动画的属性

Learning CoreData:建立sqlite文件

基本流程

// 1. 找目录
- (NSString *)applicationDocumentsDirectory {
    if (debug==1) {
        NSLog(@"Running %@ '%@'", self.class,NSStringFromSelector(_cmd));
    }
    return [NSSearchPathForDirectoriesInDomains(NSDocumentDirectory, NSUserDomainMask,YES) lastObject];
}


// 2. 建新目录
- (NSURL *)applicationStoresDirectory {
    if (debug==1) {
        NSLog(@"Running %@ '%@'", self.class, NSStringFromSelector(_cmd));
    }

    NSURL *storesDirectory =
    [[NSURL fileURLWithPath:[self applicationDocumentsDirectory]]
     URLByAppendingPathComponent:@"Stores"];

    NSFileManager *fileManager = [NSFileManager defaultManager];
    if (![fileManager fileExistsAtPath:[storesDirectory path]]) {
        NSError *error = nil;
        if ([fileManager createDirectoryAtURL:storesDirectory
                  withIntermediateDirectories:YES
                                   attributes:nil
                                        error:&error]) {
            if (debug==1) {
                NSLog(@"Successfully created Stores directory");}
        }
        else {NSLog(@"FAILED to create Stores directory: %@", error);}
    }
    return storesDirectory;
}

API详解

  1. [NSSearchPathForDirectoriesInDomains(NSDocumentDirectory, NSUserDomainMask,YES) lastObject]

    NSUserDomainMask: mask domain. mask 就是~
    NSDocumentDirectory: document目录。
    YES:将波浪形展开。

  2. stringByAppendingPathComponent和stringByAppendingString的区别

    自我理解，不保证正确，但亲自试验确实可行：若用 stringByAppendingString 则需要手动在名称前加 “/”符号，而stringByAppendingPathComponent则不需要，它会自动添加

  3. NSURL *storesDirectory = [NSURL fileURLWithPath:[documentPath stringByAppendingPathComponent:@"Store"]];不会创建新目录，只是给出一个链接。真正创建目录的是
    `[fileManager createDirectoryAtURL:storesDirectory

    withIntermediateDirectories:YES
                 attributes:nil
                      error:&error]`
