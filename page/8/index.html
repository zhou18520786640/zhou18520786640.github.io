OC设计模式之简单工厂模式

发表于 2016-03-05

简单工厂模式概述

简单工厂模式（简称工厂模式）属于创建型模式，又叫做静态工厂方法，也就是说使用这种模式是为了创建对象。

角色

简单工厂模式中有几种类型的角色:
工厂角色（Factory）:创建具体产品的类，负责创建的逻辑
抽象产品（Product）角色: 为具体产品提供统一接口,可以是协议，也可以是抽象类
具体产品（Concrete Product):角色要创建的对象的对应的类
客户端: 调用工厂角色创建具体产品。
显然，工厂角色的创建是这种模式的核心。下面是示意图

演示代码

地址：https://github.com/zhou18520786640/Factory

Factory 类

#import "ZZProduct.h"

@interface ZZProductFactory : ZZProduct
+ (ZZProduct *)createProductWithclassName:(NSString *)className;
@end

#import "ZZProductFactory.h"

@implementation ZZProductFactory
+ (ZZProduct *)createProductWithclassName:(NSString *)className {
    Class class = NSClassFromString(className);
    ZZProduct *product = [[class alloc] init];
    return product;
}
@end

使用场景

在下面一些场景下可以考虑使用简单工厂模式

  1. 需要创建具体产品的地方很多，如果不使用工厂类创建，每个客户端地方都需要｀if(…)else(…)`来创建.代码不优雅。实际上这部分逻辑可以全部封装到工厂类来实现
  2. 创建每一个具体产品后还需要做一些繁琐的配置工作。在客户端做这些配置工作也会让代码很难看。
  3. 1还有一个问题，就是又新添加一个具体产品的时候。如果客户端有很多地方，需要很多地方都继续添加else(…)创建具体的产品。

补充

在iOS中，可以根据类名（NSString）生成对应的Class对象。然后根据Class对象生成对应的实例对象
应用这种方法，可以避免在工厂类中写if…else.

Class class = NSClassFromString(className);
ZZProduct *product = [[class alloc] init];
