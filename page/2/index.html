。选择云应用。
。
然后进入控制台。添加应用即可。注意选择部署环境为python.配置如下

部署python代码

编辑应用代码

进入我们的应用，使用git作为版本部署工具。实现一个简单的helloword程序。我们的思路是在本地创建我们的代码工程。然后push到新浪云应用中去。 这里本地我们只需要编写两个文件 config.yaml index.wsgi。分别作为应用程序的配置文件和程序入口文件

config.yaml文件内容如下

name: helloworld
version: 1

index.wsgi文件如下

import sae

def app(environ, start_response):
    status = '200 OK'
    response_headers = [('Content-type', 'text/plain')]
    start_response(status, response_headers)
    return ['Hello, world!']

application = sae.create_wsgi_app(app)

将这两个文件放在一个文件夹下（如helloword）.整个文件的编写就完成了。

部署应用

我们选择Git作为部署工具。

首先，创建一个Git代码仓库，将刚才的代码提交到Git仓库中。

jaime@westeros:~/helloworld$ git init
jaime@westeros:~/helloworld$ git add .
jaime@westeros:~/helloworld$ git commit -m 'Initial commit'

然后，添加一个新的Git远程仓库sae，地址为： https://git.sinacloud.com/应用名 ，然后将刚才提交的代码推到远程Git仓库的分支1中。

jaime@westeros:~/helloworld$ git remote add sae https://git.sinacloud.com/helloworld
jaime@westeros:~/helloworld$ git push sae master:1

这时，需要输入帐号和密码，千万注意不是输入新浪的帐号和密码，而是安全帐号邮箱和安全帐号密码,如果不知道，可以进入用户中心的自己的帐号下查询
至此，整个代码的部署就完成了，在浏览器中输入 http://helloword.applinzi.com,注意不是官网说的 http://helloworld.sinaapp.com ,就可以访问刚提交的应用了。

UITableView的一些注意事项

发表于 2016-04-09

实现cell之间的间距

有两种常见的方案。
1.利用section.不同的section之间可以设置footerView和headerView。通过设置footerView和headerView来实现间距
2.将cell的背景色设置比要展示的内容高。那样看起来就有部分间距。

不要在UITableView中嵌套UITableView。

理论上说，UITableView中是可以嵌套UITableView的。但是实际上这样做会产生很多问题。

TableView的Header不滚动出显示区域

当设置plain风格的时候，Header不会和cell一样滚出屏幕显示区域。设置group风格即可。

self.tableView = [[UITableView alloc] initWithFrame:self.view.bounds style:UITableViewStyleGrouped];

隐式动画

发表于 2016-03-26

什么是隐式动画

“所谓的隐式动画。之所以叫隐式是因为我们并没有指定任何动画的类型。我们仅仅改变了一个属性，然后Core Animation来决定如何并且何时去做动画”

通俗的说，就是只需要改变layer的属性。CoreAnimation库自动帮你实现动画。相比于你考虑如何做一个动画，还不如考虑如何关闭动画。因为动画是不需要配置的，不要动画却需要配置。下面是一个简单的demo,点击按钮后，layer的颜色经过一定时间动画后从蓝色变为红色。

#import "ZZViewController.h"

@interface ZZViewController ()
@property (nonatomic, strong) CALayer *layer;
@end

@implementation ZZViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
    [button addTarget:self action:@selector(buttonDidPressed:) forControlEvents:UIControlEventTouchUpInside];
    button.frame = CGRectMake(40, 40, 80, 80);
    button.backgroundColor = [UIColor redColor];
    [self.view addSubview:button];

    CALayer *layer = [CALayer layer];
    self.layer = layer;
    self.layer.backgroundColor = [UIColor blueColor].CGColor;
    self.layer.frame = CGRectMake(120, 120, 120, 120);
    [self.view.layer addSublayer:self.layer];

}

- (void)buttonDidPressed:(UIButton *)button {
    self.layer.backgroundColor = [UIColor redColor].CGColor;

}

@end

决定隐式动画执行时间的事务 CATransaction

“但当你改变一个属性，Core Animation是如何判断动画类型和持续时间的呢？实际上动画执行的时间取决于当前事务的设置，动画类型取决于图层行为”

在上面那个Demo中，有一个从蓝色到红色过度的动画。但是这远远不够，我们需要能够设置动画的执行时间。事务类可以实现。

“事务是通过CATransaction类来做管理，这个类的设计有些奇怪，不像你从它的命名预期的那样去管理一个简单的事务，而是管理了一叠你不能访问的事务。CATransaction没有属性或者实例方法，并且也不能用+alloc和-init方法创建它。但是可以用+begin和+commit分别来入栈或者出栈。”
