发表于 2016-03-26

什么是隐式动画

“所谓的隐式动画。之所以叫隐式是因为我们并没有指定任何动画的类型。我们仅仅改变了一个属性，然后Core Animation来决定如何并且何时去做动画”

通俗的说，就是只需要改变layer的属性。CoreAnimation库自动帮你实现动画。相比于你考虑如何做一个动画，还不如考虑如何关闭动画。因为动画是不需要配置的，不要动画却需要配置。下面是一个简单的demo,点击按钮后，layer的颜色经过一定时间动画后从蓝色变为红色。

#import "ZZViewController.h"

@interface ZZViewController ()
@property (nonatomic, strong) CALayer *layer;
@end

@implementation ZZViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    UIButton *button = [UIButton buttonWithType:UIButtonTypeCustom];
    [button addTarget:self action:@selector(buttonDidPressed:) forControlEvents:UIControlEventTouchUpInside];
    button.frame = CGRectMake(40, 40, 80, 80);
    button.backgroundColor = [UIColor redColor];
    [self.view addSubview:button];

    CALayer *layer = [CALayer layer];
    self.layer = layer;
    self.layer.backgroundColor = [UIColor blueColor].CGColor;
    self.layer.frame = CGRectMake(120, 120, 120, 120);
    [self.view.layer addSublayer:self.layer];

}

- (void)buttonDidPressed:(UIButton *)button {
    self.layer.backgroundColor = [UIColor redColor].CGColor;

}

@end

决定隐式动画执行时间的事务 CATransaction

“但当你改变一个属性，Core Animation是如何判断动画类型和持续时间的呢？实际上动画执行的时间取决于当前事务的设置，动画类型取决于图层行为”

在上面那个Demo中，有一个从蓝色到红色过度的动画。但是这远远不够，我们需要能够设置动画的执行时间。事务类可以实现。

“事务是通过CATransaction类来做管理，这个类的设计有些奇怪，不像你从它的命名预期的那样去管理一个简单的事务，而是管理了一叠你不能访问的事务。CATransaction没有属性或者实例方法，并且也不能用+alloc和-init方法创建它。但是可以用+begin和+commit分别来入栈或者出栈。”
