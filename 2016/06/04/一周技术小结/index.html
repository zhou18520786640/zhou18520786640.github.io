一周实践小结

发表于 2016-06-04

主动实践的被动学习

在实际工作中会遇到一些知识的盲点和积累一些经验，这里做一点简单的总结。

不要试图使用button.titleLabel.text来读写button的显示内容？

我们都知道使用button要显示一个值的时候。不能类似cell一样调用cell.textLabel.text = @"123123";来调用
button.titleLabel.text = @"rrr"(虽然有时候有效),正确的写的方法应该是 [button setTitle:@"我是按钮" forState:UIControlStateNormal];.

同时读的操作也不该使用button.titleLabel.text来获取其显示内容。[button titleForState:UIControlStateNormal]是更加稳妥的方案。 button.titleLabel.<#property#>是用于设置title的颜色和字体大小等。如button.titleLabel.textColor等。

手写collectionview一定要在初始化对象的时候传入UICollectionViewFlowLayout对象

只有在iOS8下才可以使用自动布局+xib下来自动布局tableViewcell

拖入xib后的控件视图宽度不一定等于屏幕宽度？

拖入xib后的视图控件的宽度不一定等于当前屏幕的宽度。
需求场景如下。我现在有一个ImageView 几个UILabel.放置顺序是从上到下。imageView对齐。其他Label和imageView左对齐。我设定UIImageView居中后。只是把Label放在和uiimageview同样x的位置。然后设定它们与父视图的位置的约束。因为屏幕的宽度和xib view宽度不一致。程序运行后，label还是保持原来的位置。uiimageview为了居中，会向左偏移。实际上uiimageview的位置才是我真正需要的位置。我应该设定uiimageview后。其他label都和uiimageview左对齐才对。并且保险的方案是将xib中的根视图的宽度调节到一致。

为什么RAC中叫做双向绑定而不是双向赋值？

因为rac中有一个响应式概念。对一个对象的属性做持续的观察而并不是简简单单的只是一次赋值操作
如命令式编程中
label.text = model.text; // model的text再赋予新的值不会影响label的显示值。
rac中对model的属性进行监听。然后将信号的输出反馈给label，就是绑定model的属性。因为model的text在model生命周期内任何一次变化都会反馈给label.而双向绑定就是说身为视图角色的view的属性也可以被监听。

xib和自动布局不适用一些场景？

复杂的动画不适用于。cell的高度计算在iOS8以下。UITableViewCell,UICollectionViewCell的变动很剧烈的场景。复杂的布局场景不太适用于。复杂的视图层次也不利于调试。

如果工时的估计还缺乏必要的关键条件。可以给PMO说一下要等会儿再给他答复。

自定义导航条的一些方法？

1.可以隐藏系统的导航条。然后贴一个自己的view,注意要在合适的时机隐藏或者显示导航条。
2.使用运行时。这个专门写一篇文章分析

简单的东西一学就会，复杂的东西一学不会？

简单的东西我一学就会，一些小技巧，小tips用的得心应手。但是这些都是术的范畴。对于复杂的东西，如架构设计 运行时之类的每一次只是浅尝一下就放在一边了。这种可以上升到艺的东西还是按照学习小tip的方法来实践显然是没有效果的的。要花时间来实践提高我自己艺的学习方法。现在暂时想到的方法步骤是1.看国内demo 2.看国内教程 3.实践写玩具应用，同时不断google 4.看官方文档 5.在1，2，3，4过程中不断问老司机。6.在1，2，3，4，5中不断写个人技术文档 7.把任务或者分阶段来设定目标

向别人学习学习其学习方法比学习知识更能受益

学习别人的学习方法，如果适用自己，就果断使用。
